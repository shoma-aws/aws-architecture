# 概要
CloudWatch Logs、CloudWatch Metrics、CloudTrailから取得した情報を統合し、  Bedrock（Claude）に自然言語として渡すことで、  インフラ障害の原因を自動分析するシステムを構築しました。

# 作成理由
業務の中で、障害が発生した際にはまずログやメトリクスを確認し原因調査を行いますが、<br>
- 調査する範囲が担当者によって異なる
- 情報のまとめ方が人によってばらつく<br>
といった課題があり、原因特定のスピードや精度に差が出ると感じました。<br>
また、アプリケーションチームが利用しているサーバーで障害が発生した場合、
インフラチームによる調査を待つ必要があり、　
- 調査開始までに時間がかかる可能性がある
- 初動対応が遅れるリスクがある<br>
という運用上の課題もあると考えました。<br>
これらの背景から、ログ・メトリクス・操作履歴を統合し、AIによって原因分析を自動化することで、<br>
「調査の属人化解消」と「初動対応の迅速化」を目的として本システムを構築しました。

# 使用サービス
- Amazon EC2
- Amazon CloudWatch Logs
- Amazon CloudWatch Metrics
- Amazon CloudWatch Alarm
- AWS CloudTrail
- AWS Lambda
- Amazon Bedrock（Claude）

# 処理フロー

1. EC2でアプリケーションが動作<br>
2. CloudWatch Logsにログが蓄積される<br>
3. CloudWatch MetricsでCPUなどのメトリクスを取得<br>
4. CloudWatch Alarmが閾値超過を検知<br>
5. Lambdaが自動起動<br>
6. Lambdaが以下の情報を取得<br>
   - ログ（CloudWatch Logs）
   - CPU使用率（CloudWatch Metrics）
   - 操作履歴（CloudTrail）<br>
7. Lambda内でデータを整形<br>
8. AIに渡すため自然言語へ変換<br>
9. Bedrock（Claude）で原因分析<br>
10. 分析結果をCloudWatch Logsに出力
