
# 障害対応シナリオ

本ディレクトリでは、Terraform で構築した AWS 環境を用いて、
運用・障害対応の練習として実施したシナリオを整理しています。

## 前提構成
- VPC / Public & Private Subnet
- ALB + Listener + Target Group
- Auto Scaling Group 配下の EC2
- CloudWatch による監視

## シナリオ一覧
1. Security Group 誤設定による Web アクセス不可
2. SSH 接続不可
3. サービスは起動しているがアクセス不可
4. CPU 高騰によるレスポンス悪化

## 課金事故
AWSの学習環境構築中に意図しない高額課金が発生したため、
原因を特定し不要なリソースを停止・削除する対応を実施。
おそらくNATGeteway、EC2、ElasticIPかと、、
この経験から、クラウド環境におけるコスト管理の重要性と、
インシデント発生時の初動対応の重要性を学んだ。
